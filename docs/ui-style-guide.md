# 深夜匿名文字空间 — UI 设计规范

> 产品气质：深夜 · 安静 · 克制 · 孤独感 · 陪伴感  
> 设计原则：少即是多，暗即是亮，慢即是快  

---

## 一、配色方案

### 1.1 主色调

| Token | 色值 | 用途 |
|-------|------|------|
| 背景色 | `#0A0A0A` | 全局底色，略暖于纯黑，像深夜天空 |
| 文字主色 | `rgba(255,255,255,0.50)` | 正文内容，克制亮度，不刺眼 |
| 文字辅色 | `rgba(255,255,255,0.28)` | 提示语、时间标签、次要信息 |
| 文字弱色 | `rgba(255,255,255,0.12)` | 字数统计、占位符、极边缘信息 |
| 强调暖色 | `#BFA378` | 唯一"亮"色，像远方路灯 / 烛火 |
| 强调暖色半透 | `rgba(191,163,120,0.50)` | 焦点状态（光标、选中标识） |

### 1.2 边框 & 分割

| Token | 色值 | 用途 |
|-------|------|------|
| 极隐边框 | `rgba(255,255,255,0.06)` | 输入框边缘、卡片间隐约分界 |
| 可见边框 | `rgba(255,255,255,0.15)` | 按钮描边、可交互元素边界 |
| 分割符号 | `rgba(255,255,255,0.10)` | 文字间 · · · 分割线 |

### 1.3 功能色

| 场景 | 色值 | 说明 |
|------|------|------|
| 在线人数指示 | `#BFA378` 或 `rgba(191,163,120,0.70)` | 微小的暖色圆点 |
| 发布成功提示 | `rgba(191,163,120,0.60)` | 短暂浮现，随即消失 |
| 错误 / 空内容 | `rgba(255,255,255,0.20)` | 不搞红色警告，太刺眼；用"更暗"表达不可用 |

### 1.4 配色铁律

- **禁止纯白 `#FFFFFF`** — 任何场景下文字/元素都不使用纯白
- **禁止纯黑 `#000000`** — 底色带微弱暖意或蓝意
- **禁止高饱和色** — 不出现红、绿、蓝、黄等鲜艳颜色
- **强调暖色仅用于"微光"** — 小面积、低不透明度、不抢眼
- **一切颜色都像被夜色稀释过**

## 二、字体规范

### 2.1 字体栈

```css
font-family:
  -apple-system,            /* iOS: PingFang SC */
  BlinkMacSystemFont,
  "Helvetica Neue",
  "PingFang SC",
  "Noto Sans SC",           /* Android 备选 */
  sans-serif;
```

> 微信小程序无法引入自定义字体，依赖系统默认中文字体。  
> 不刻意指定宋体/楷体等风格化字体 — 系统默认就是最"安静"的选择。

### 2.2 字号 & 字重

| 层级 | 字号 (rpx) | 字号 (px) | 字重 | 用途 |
|------|-----------|----------|------|------|
| 正文 | 34 rpx | ~17px | 300 (Light) | 匿名文字卡片内容 |
| 辅助正文 | 28 rpx | ~14px | 300 | 书写页输入区 |
| 提示语 | 26 rpx | ~13px | 400 (Regular) | 页面引导文字、按钮 |
| 时间标签 | 22 rpx | ~11px | 300 | "深夜 3:12"、在线人数 |
| 微小文字 | 20 rpx | ~10px | 300 | 字数统计、页脚 |

### 2.3 排版规则

- **行高**：正文 1.8~2.0，给文字呼吸感；辅助文字 1.6
- **字间距**：正文 `letter-spacing: 0.5px`，略微松散，放慢阅读节奏
- **对齐**：全部左对齐，不居中 — 居中太"设计感"，左对齐更像深夜独白
- **每行字数**：控制在 18~22 个汉字以内（约屏幕宽度 70%），两侧大量留白

## 三、留白规则

### 3.1 核心理念

> 空白不是"没东西"，空白是产品的一部分。  
> 深夜的屏幕应该是一片黑暗，文字像浮在黑暗中的微光。  
> 任何时候，屏幕 **40%~60% 的区域是纯黑色空白**。

### 3.2 间距尺度

| 间距 | rpx | 场景 |
|------|-----|------|
| **XS** | 8 rpx | 极小元素间距（标签内） |
| **S** | 16 rpx | 相关元素间距 |
| **M** | 32 rpx | 组件内间距 |
| **L** | 48 rpx | 卡片之间 |
| **XL** | 80 rpx | 文字卡片之间（竖直方向） |
| **XXL** | 120 rpx | 内容区块上下留白 |

### 3.3 页面级留白

```
┌────────────────────────────────┐
│                                │  ← 120rpx 顶部留白
│  "睡不着，看了两集..."          │
│                                │  ← 80rpx
│            ·                    │
│            ·                    │  ← 分割符号 + 上下各 40rpx
│            ·                    │
│  "在想一个人，很久没联系了..."   │
│                                │  ← 80rpx
│            ·                    │
│            ·                    │
│            ·                    │
│  "明天面试，又焦虑又期待..."    │
│                                │
│                                │  ← 底部大量留白
│         [ 写下心事 ]            │  ← 按钮浮在底部上方
│                                │
└────────────────────────────────┘
```

### 3.4 留白铁律

- 卡片之间必有 80rpx 以上间隙，**宁可翻屏不堆砌**
- 页面顶部留白 ≥ 120rpx（包含状态栏高度）
- 文字行宽不超过屏幕 70%，两侧自然留白
- 分割符号 `· · ·` 居中，上下各 40rpx 空白
- 同一屏最多展示 2~3 条文字卡片

## 四、按钮风格

### 4.1 按钮类型（MVP 只有两种）

#### 主按钮 —「幽灵按钮」

```
┌──────────────────────────────┐
│         写下心事               │  ← 文字辅色，字重 400
└──────────────────────────────┘
  描边：rgba(255,255,255,0.15)
  背景：透明
  圆角：4rpx（接近直角，冷静）
  宽度：自适应文字 + 水平 64rpx padding
  高度：80rpx
  字号：26rpx
```

**交互状态**：
- 默认：低调暗色描边，文字半透明
- 可点击：光标接近时描边亮度微微提升
- 禁用：描边和文字降到 0.08 不透明度，接近不可见
- 点击反馈：描边短暂变亮，同时按钮区域轻微呼吸缩放（scale 0.97 → 1.0，200ms）

#### 次级按钮 —「纯文字按钮」

```
← 返回                        扔掉 →
```

- 无描边、无背景
- 文字弱色或辅色
- 点击时文字短暂变亮 0.2s
- 用于导航、放弃编辑等低优先级操作

### 4.2 按钮文案风格

| 场景 | 文案 | 说明 |
|------|------|------|
| 进入书写 | "写下心事" | 不用"发布""发帖"，太功能化 |
| 提交发布 | "扔进夜色" | 不用"发送""提交"，深夜感语境 |
| 返回 | "←" 或 "返回" | 极轻极淡，不要 big 箭头 |
| 放弃 | "扔掉" | 不用"取消""删除"，与匿名气质匹配 |

### 4.3 按钮铁律

- **不做实心按钮** — 太亮，打破深夜感
- **不做大圆角** — 圆角 = 友好 = 白天产品
- **不做阴影** — 深夜无光源，没有投影
- **不做大按钮** — 按钮不应成为视觉焦点，文字才重要

## 五、动效节奏

### 5.1 时间尺度定义

| 节奏 | 时长 | 感觉 | 场景 |
|------|------|------|------|
| **极慢** | 2000~3000ms | 像呼吸一样慢 | 卡片首次浮现 |
| **慢** | 800~1200ms | 从容不迫 | 卡片错开入场、页面过渡 |
| **缓** | 400~600ms | 不慌不忙 | 按钮状态切换、元素显隐 |
| **快** | 150~200ms | 即时但不唐突 | 点击反馈、toast 出现 |

### 5.2 核心动效

#### 文字卡片浮现（最重要）

```
opacity:  0 → 1       (duration: 2400ms, easing: ease-out)
transform: translateY(12rpx) → translateY(0)  (duration: 2400ms)
```

- 每张卡片间隔 800ms 依次出现
- 第一张卡片延迟 400ms 后开始
- 新帖子插入列表顶部时，先占位（空 80rpx），再浮现文字

#### 页面切换

```
交叉淡入淡出: opacity 0→1, duration: 400ms
无滑动、无方向感 — 深夜不需要"从哪来到哪去"的空间隐喻
```

#### 按钮点击反馈

```
scale: 1 → 0.97 → 1  (duration: 200ms, easing: ease-out-back 轻微回弹)
描边不透明度: 0.15 → 0.30 → 0.15
```

#### 发布成功提示

```
文字 "已投入夜色中" 浮现：
opacity: 0 → 0.6 → 0  (duration: 2000ms total)
无 toast 弹窗，纯文字在屏幕中下方浮现后自动消失
```

#### 加载呼吸

```
呼吸圆点：
opacity: 0.2 → 0.5 → 0.2  (cycle: 2000ms, infinite)
单个小圆点，居中，像远处未眠的灯火
```

#### 输入框光标

```
闪烁周期: 1500ms（比常规光标慢一倍）
颜色: rgba(191,163,120,0.50)
```

### 5.3 动效铁律

- **所有动画 easing 用 `ease-out` 或 `linear`** — 禁止 bounce、elastic、ease-in-out
- **不跟随手指** — 不用跟手拖拽、不用回弹滚动特效
- **不抢节奏** — 动效是"让用户慢下来"，不是"让页面活起来"
- **可被中断** — 用户滚动时新卡片停止动画，直接显示（尊重用户节奏）
- **动效只加在卡片入场** — 列表滚动、页面切换不加多余动效

## 六、文字卡片样式（"聊天气泡"的深夜版本）

### 6.1 卡片结构

```
┌─────────────────────────────────────────────┐
│                                              │
│  ┊                                           │  ← 极隐的 1px 左边线
│  ┊  "睡不着，翻来覆去地想了很久。            │
│  ┊   有些事情，白天想不明白，                  │
│  ┊   深夜反而清楚了。"                        │
│  ┊                                           │
│                                              │
│                      · · ·                    │  ← 卡片下方分割
│                                              │
│          （下一张卡片在上方 80rpx 处）         │
└─────────────────────────────────────────────┘
```

### 6.2 卡片参数

| 属性 | 值 | 说明 |
|------|-----|------|
| 背景 | 透明 | 卡片不设底色，文字直接漂浮在黑色背景上 |
| 描边 | 无 | 不设四周边框 |
| 左侧线 | `1px solid rgba(255,255,255,0.06)` | 唯一的结构装饰，极隐 |
| 宽度 | 屏幕 70% | 右半侧大片留白 |
| 内边距 | `0 0 0 24rpx` | 左侧线与文字之间的小间隙 |
| 圆角 | 无 | 直角，不友好，不亲切 |
| 对齐 | 左对齐 | 就像深夜备忘录里随手打下的字 |

### 6.3 卡片间分割

```
              · · ·
```

- 三个点，居中
- 颜色 `rgba(255,255,255,0.10)`
- 上下各 40rpx 空白
- 字号 20rpx
- 代表"时间的流逝"而非"内容的分割"

### 6.4 特殊状态

#### 新卡片刚刚抵达

```
先空出 80rpx 的位置（布局瞬间撑开）
→ 400ms 后开始 opacity 0→1（文字浮现）
→ 2400ms 后完全可见
```

#### 旧卡片即将过期（未来迭代）

```
不显示倒计时、不显示剩余时间
只是某一天它不再出现
— 消失得和到来时一样安静
```

## 七、页面气质建议

### 7.1 首页 —「夜色中的微光」

- 打开瞬间不要有任何加载动画 — 直接黑屏，然后第一张卡片开始浮现
- 每张卡片都是一段独立的内心独白，卡片间距大到像两个互不认识的人在黑暗中擦肩
- 顶部时间标签用极小的字："深夜 3:12"，偶尔变成"夜深了""天快亮了"
- 在线人数不展示数字，展示为"有一些人和你一样醒着"
- 页面像是一扇窗，不是信息流

### 7.2 书写页 —「对夜色说话」

- 进入后，键盘自动弹起，光标开始缓慢闪烁
- 页面只有三样东西：提示语"此刻，你在想什么？"、输入区、字数计数
- 键盘收起时，输入的文字像是刻在黑色屏幕上，无边界框
- 发布按钮在键盘弹起时贴键盘上方，键盘收起时沉到底部
- 发布后不弹出任何确认框 — 文字消失，页面返回，像是把心事扔进了夜色里
- 如果什么都不写就返回，不保存、不提醒、不留痕迹 — 匿名感的完整闭环

### 7.3 通用气质

- **不对称** — 文字偏左、右侧空着，制造孤独感
- **无品牌感** — 不出现 logo、名称、slogan，产品不需要自我介绍
- **无社交感** — 没有头像、昵称、点赞数、评论数、转发按钮
- **时间感** — 唯一动态元素是时间标签，强化"深夜"语境
- **一致性** — 每个像素都在说同一句话：这里是深夜，不需要大声
