# 深夜感优化方案

## 一、UI 优化方案

### 1. 首页深夜文案系统

**现状**：`MOOD_TEXTS` 7条随机取一条，不区分时段。

**改进**：按小时分槽位，每个时段 3-5 条，更精准匹配凌晨情绪。

```
22:00–23:59  深夜   初入夜晚，情绪还带着白天的余温
00:00–01:59  深夜   失眠中段，最清醒也最孤独
02:00–04:59  凌晨   夜最深，话语最轻
05:00–06:59  天快亮了  微光将至，夜晚收尾
```

每条文案保持克制，不鸡汤、不共鸣绑架、不刻意温暖。用陈述句或不完整句子。

### 2. 空聊天室提示语

**现状**：无。进入后一片空白。

**改进**：消息列表为空时，中间显示一行淡字。

不要用「说点什么吧」「开始聊天」等呼唤行动文案。用安静陈述，让用户自己决定要不要开口。

### 3. 输入框 placeholder 文案

**现状**：`说点什么...`

**改进**：低调，偏孤独感，不引导、不鼓励、不社交化。

方向：`想说就写点什么...` 或其他更轻的变体。核心是不催促。

### 4. 聊天页时间氛围

**现状**：顶栏显示 `深夜 3:42`，进入后不变。

**改进**：顶栏时间每分钟更新，凌晨时段文案自动切换。页面 onShow 时刷新。

### 5. 黑色UI细节

**现状**：纯黑 `#0A0A0A`，白字透明度 0.12~0.50，金色 `#BFA378`。

**改进**：
- 背景不要纯黑，用带微弱冷暖偏移的黑色（深夜偏蓝黑，凌晨偏暖黑），页面间过渡自然
- 金色降饱和，从 `#BFA378` 降到 `#8A7A6A` 或更低，像远处余烬而非金子
- 分割线、边框透明度再低一点，0.04~0.06 级别，若隐若现即可

### 6. 消息间距与留白

**现状**：消息间距 `80rpx`，左右留白 `64rpx`。

**改进**：
- 消息间距增大到 `100-120rpx`，稀疏感让每条消息更像独白
- 消息列表顶部增加留白，第一条消息不要贴顶
- 底部留白大于输入栏高度，避免最后一条消息被输入栏紧逼

### 7. 慢节奏感

**现状**：消息入场动画 `2.4s`（已经偏慢）。

**改进**：
- 保持现有动画时长
- 首页呼吸点动画保持 `2.4s`
- 不做打字机效果、不做loading、不做过渡动画
- 页面切换用默认导航动画（已经够用）
- "慢"来自留白和间距，不来自动效

### 8. 深夜在线人数显示

**现状**：`getPresenceCount()` 返回 2~12 随机数，首页显示「N人和你一样醒着」。

**改进**：接入真实 WebSocket 在线人数。服务端已有 `wss.clients.size`，通过 WS 广播给所有客户端。首页显示真实数字。

不显示头像、不显示列表、不显示谁是谁。只显示一个数字。

### 9. 渐变背景

**现状**：纯黑 `#0A0A0A` 无渐变。

**改进**：在页面顶部或底部加极其轻微的光晕，模拟月光或城市余光。

技术：一个 fixed 径向渐变圆形，透明度 0.03~0.05，作为氛围层。不可明显看到，不刻意强调。感觉不到它存在，但去掉会冷。

### 10. 凌晨时间动态变化

**现状**：22:00~06:00 分三个时段，时间段切换后文案不动（需重新进入页面）。

**改进**：
- 首页 `onShow` 时刷新时段、文案、在线人数
- 文字颜色根据时段微调：深夜段偏冷白，凌晨段偏暖白，天亮段接近冷灰
- 不做时间计数器、不做倒计时

## 二、文案方案

### 首页时段文案

```
22:00-23:59 深夜
  · 睡不着的人，不止你一个。
  · 夜晚是白天的找零。
  · 把今天没说完的话，带到夜里。
  · 此刻醒着，也不用跟谁解释。

00:00-01:59 深夜
  · 失眠不是惩罚，是独处。
  · 凌晨两点前，都还算今天。
  · 不睡，是不舍得把今天交出去。
  · 有些话，只能在深夜说。

02:00-04:59 凌晨
  · 夜到深处，声音反而轻了。
  · 全世界都黑屏了。
  · 没有人在这个时间打广告。
  · 这是夜晚的底部。
  · 快天亮了。再坐一会儿。

05:00-06:59 天快亮了
  · 天边开始变色了。
  · 又陪了夜晚一夜。
  · 天亮了。晚安。
```

### 空聊天室提示

```
还没有人开口。
```

### 输入框 placeholder

```
写点什么...
```

## 三、页面气质建议

**调色板**

| 角色 | 色值 | 说明 |
|------|------|------|
| 背景 | `#08090D` | 深夜蓝黑，不再纯中性黑 |
| 背景光晕 | `#1A1A2E` 径向渐变 | 页面顶部微光，透明度 0.04 |
| 主文字 | `rgba(255,255,255,0.45)` | 保持现有 |
| 辅助文字 | `rgba(255,255,255,0.20)` | 维持微妙层级 |
| 强调/在线点 | `#6B5E52` | 降饱和，余烬感 |
| 分割线 | `rgba(255,255,255,0.04)` | 若隐若现 |

**字体层级**

| 层级 | 大小 | 使用 |
|------|------|------|
| 时间数字 | 80rpx | 首页时钟 |
| 时段标签 | 26rpx | 首页时段 |
| 文案 | 34rpx | 首页 mood |
| 辅助信息 | 22rpx | 在线人数 |
| 消息正文 | 30rpx | 聊天内容 |
| 消息时间 | 20rpx | 消息时间戳 |
| 顶栏 | 22rpx | 聊天页时段 |
| placeholder | 28rpx | 输入框 |

**整体气质**

- 不是"陪你度过失眠"——不陪伴、不疗愈
- 不是"你是特别的"——不肯定、不安慰
- 是"夜晚本身"——中立、安静、如其所是
- 像一个凌晨3点的便利店：存在，但不打扰

## 四、技术影响

| 改动 | 涉及文件 | 复杂度 |
|------|---------|--------|
| 首页文案分时段 | `utils/time.js` | 低 |
| 空聊天室提示 | `pages/chat/chat.wxml` | 低 |
| placeholder 文案 | `pages/chat/chat.wxml` | 低 |
| 顶栏时间动态更新 | `pages/chat/chat.js` | 低 |
| 黑色调微调 | `app.wxss` + 各页面 wxss | 低 |
| 消息间距 | `pages/chat/chat.wxss` | 低 |
| 背景光晕 | `app.wxss` | 低 |
| 真实在线人数 | `utils/websocket.js` + `server/server.js` | 中 |
| 金色降饱和 | 各 wxss 中硬编码值 | 低 |

唯一有技术改动的是真实在线人数——服务端广播 `clients.size`，客户端接收更新。其余全是 CSS 和文案替换。
